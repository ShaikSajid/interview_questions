Q1 Event Loop Basics Beginner
▼

🎯 Core Concept

  • Single-threaded architecture
  • Non-blocking I/O operations
  • Event-driven programming
  • Call stack and callback queue

⚡ Key Benefits

  • Handles thousands of connections
  • Low memory footprint
  • No context switching overhead
  • Efficient for I/O-heavy apps

🏦 Banking Use Case

  • Process transaction queue
  • Handle concurrent requests
  • Real-time notifications
  • API gateway processing
💻 Example: Non-Blocking I/O
▼ 
// ❌ BAD: Blocking (Synchronous)
const fs = require('fs');

console.log('Start');
const data = fs.readFileSync('large-file.txt');
console.log('File read complete');
console.log('End');

// Output: Start → File read complete → End
// Problem: Blocks for entire file read (5 seconds!)

// ✅ GOOD: Non-Blocking (Asynchronous)
console.log('Start');
fs.readFile('large-file.txt', (err, data) => {
  console.log('File read complete');
});
console.log('End');

// Output: Start → End → File read complete
// Benefit: Doesn't block! Other operations continue

🔑 Key Takeaways

  • Single thread handles all requests via event loop
  • Non-blocking I/O allows concurrent operations
  • Callbacks execute when I/O completes
  • Best for I/O-heavy applications (APIs, real-time apps)
  • Not ideal for CPU-intensive tasks (use worker threads)

✅ DO's

  • Use async/await or promises for I/O operations
  • Leverage Node.js for I/O-bound tasks (APIs, databases)
  • Use worker threads for CPU-intensive operations
  • Monitor event loop lag in production
  • Design APIs with non-blocking patterns

❌ DON'Ts

  • Don't perform blocking operations (long loops) in main thread
  • Don't use sync methods (fs.readFileSync) in production
  • Don't run heavy computations without worker threads
  • Don't ignore event loop blocking warnings
  • Don't use Node.js for pure CPU tasks without optimization
Q2 Event Loop Phases Intermediate
▼

⏰ Phase 1: Timers

  • setTimeout callbacks
  • setInterval callbacks
  • Executes after threshold

📋 Phase 2: Pending Callbacks

  • Deferred I/O callbacks
  • TCP errors
  • System operations

🔄 Phase 3: Poll

  • Retrieve new I/O events
  • Execute I/O callbacks
  • Most time spent here

✅ Phase 4: Check

  • setImmediate callbacks
  • Runs after poll phase
  • Higher priority than timers

🚫 Phase 5: Close Callbacks

  • socket.on('close')
  • Cleanup operations
  • Resource deallocation

⚡ Microtasks

  • process.nextTick()
  • Promise callbacks
  • Runs between phases
💻 Example: Phase Execution Order
▼ 
// Demonstrating execution order across phases
setTimeout(() => {
  console.log('1. setTimeout (Timers phase)');
}, 0);

setImmediate(() => {
  console.log('2. setImmediate (Check phase)');
});

process.nextTick(() => {
  console.log('3. process.nextTick (Microtask)');
});

Promise.resolve().then(() => {
  console.log('4. Promise (Microtask)');
});

console.log('5. Synchronous code');

// Output order:
// 5. Synchronous code (Call stack first)
// 3. process.nextTick (Microtask - highest priority)
// 4. Promise (Microtask - after nextTick)
// 2. setImmediate (Check phase)
// 1. setTimeout (Timers phase)

// Explanation:
// 1. Sync code runs first (call stack)
// 2. Microtasks run before any phase
// 3. setImmediate usually runs before setTimeout(0)
//    when not inside I/O cycle
🏦 Banking: Payment Processing Priority
▼ 
// Priority-based payment processing using phases
const processPayment = (payment, priority) => {
  if (priority === 'urgent') {
    // Highest priority: process immediately after current operation
    process.nextTick(() => {
      console.log(`⚡ URGENT: ${payment.id} - $${payment.amount}`);
    });
  } else if (priority === 'high') {
    // High priority: process in check phase
    setImmediate(() => {
      console.log(`🔥 HIGH: ${payment.id} - $${payment.amount}`);
    });
  } else {
    // Normal priority: process after timer threshold
    setTimeout(() => {
      console.log(`📋 NORMAL: ${payment.id} - $${payment.amount}`);
    }, 0);
  }
};

// Queue payments with different priorities
processPayment({ id: 'PAY001', amount: 1000 }, 'normal');
processPayment({ id: 'PAY002', amount: 5000 }, 'urgent');
processPayment({ id: 'PAY003', amount: 2000 }, 'high');

console.log('All payments queued');

// Output:
// All payments queued
// ⚡ URGENT: PAY002 - $5000 (nextTick - first!)
// 🔥 HIGH: PAY003 - $2000 (setImmediate - second)
// 📋 NORMAL: PAY001 - $1000 (setTimeout - last)

🔑 Key Takeaways

  • 6 Phases: Timers → Pending → Poll → Check → Close
  • Microtasks execute between every phase
  • process.nextTick() has highest priority
  • setImmediate() runs before setTimeout(0) in I/O
  • Poll phase spends most time waiting for I/O

✅ DO's

  • Use setImmediate() for deferring work after I/O callbacks
  • Use process.nextTick() sparingly for highest priority tasks
  • Understand phase execution order for debugging
  • Use Promise microtasks for async flow control
  • Profile event loop phases in production

❌ DON'Ts

  • Don't overuse process.nextTick() (can starve I/O)
  • Don't use setTimeout(0) when you mean setImmediate()
  • Don't create recursive nextTick() calls (infinite loop)
  • Don't block the poll phase with long computations
  • Don't assume setTimeout() timing is exact

Q4 Streams Performance & Backpressure Advanced

📊 Memory Comparison

  • Buffer: Full file in memory
  • Stream: Fixed 64KB chunks
  • 10GB file = 10GB vs 64KB

⚠️ What is Backpressure?

  • Producer faster than consumer
  • Data backs up in memory
  • Can cause memory overflow

✅ Handling Backpressure

  • Use pipe() - automatic
  • Check write() return value
  • Listen to 'drain' event

🚀 Performance Tips

  • Adjust highWaterMark
  • Use Transform streams
  • Pipeline for error handling
💻 Example: Backpressure Problem
▼ 
const fs = require('fs');

// ❌ BAD: Ignoring backpressure (memory overflow!)
const readable = fs.createReadStream('input.txt');
const writable = fs.createWriteStream('output.txt');

readable.on('data', (chunk) => {
  // Writing without checking if buffer is full!
  writable.write(chunk); // Dangerous!
});

// Problem: If reading is faster than writing,
// chunks pile up in memory → CRASH!

// ✅ GOOD: Handling backpressure manually
readable.on('data', (chunk) => {
  const canContinue = writable.write(chunk);
  
  if (!canContinue) {
    // Buffer is full, pause reading!
    readable.pause();
    console.log('⏸️  Paused reading (backpressure)');
  }
});

writable.on('drain', () => {
  // Buffer drained, resume reading
  console.log('▶️  Resumed reading');
  readable.resume();
});

// ✅ BEST: Use pipe() - handles automatically!
readable.pipe(writable);
// pipe() manages backpressure automatically

const fs = require('fs');
const { Transform, pipeline } = require('stream');

// Generate statements for millions of customers
class StatementGenerator extends Transform {
  constructor() {
    super({ objectMode: true });
  }

  _transform(customer, encoding, callback) {
    // Generate statement for one customer
    const statement = `
=== Statement for ${customer.name} ===
Account: ${customer.accountId}
Balance: $${customer.balance}
Transactions: ${customer.transactionCount}
========================\n
`;
    
    this.push(statement);
    callback();
  }
}

// Read customer data, generate statements, write to file
const customerStream = fs.createReadStream('customers.ndjson');
const outputStream = fs.createWriteStream('statements.txt');
const generator = new StatementGenerator();

// Parse JSON lines
const jsonParser = new Transform({
  transform(chunk, encoding, callback) {
    const lines = chunk.toString().split('\n');
    lines.forEach(line => {
      if (line) {
        this.push(JSON.parse(line));
      }
    });
    callback();
  },
  objectMode: true
});

// Pipeline with automatic error handling
pipeline(
  customerStream,
  jsonParser,
  generator,
  outputStream,
  (err) => {
    if (err) {
      console.error('Pipeline failed:', err);
    } else {
      console.log('✅ Generated all statements!');
    }
  }
);

// Benefits:
// - Processes millions of customers
// - Memory: Constant ~64KB
// - Automatic backpressure handling
// - Error propagation handled

  • Backpressure: Occurs when producer > consumer speed
  • Use pipe(): Automatic backpressure management
  • Manual handling: Check write() return, listen to 'drain'

✅ DO's

  • Always handle backpressure with write() return value
  • Use pipeline() for better error handling than pipe()
  • Listen to 'drain' event before continuing writes

const encryptTransaction = (data, password) => {
  const algorithm = 'aes-256-cbc';
  
  // Generate key from password (buffer)
  const key = crypto.scryptSync(password, 'salt', 32); // 32-byte key
  const iv = crypto.randomBytes(16); // Initialization vector

  let encrypted = cipher.update(data, 'utf8', 'hex');
  encrypted += cipher.final('hex');
  
  return {
    encrypted,
    iv: iv.toString('hex') // Store IV with encrypted data
  };

💡 Error-First Callback Pattern

const fs = require('fs');

// Error-first callback pattern: (err, result) => {}
// ALWAYS check error first!

const readTransactionLog = (filename, callback) => {
  fs.readFile(filename, 'utf8', (err, data) => {
    if (err) {
      // Error occurred - pass to callback
      return callback(err, null);
    }
    
    // Success - pass result
    try {
      const transactions = JSON.parse(data);
      callback(null, transactions); // null = no error
    } catch (parseError) {
      // JSON parse error
      callback(parseError, null);
    }
  });
};

// Using the error-first callback
readTransactionLog('transactions.json', (err, transactions) => {
  if (err) {
    console.error('❌ Error reading file:', err.message);
    return;
  }

// Custom error classes for banking operations

class InsufficientFundsError extends Error {
  constructor(accountNumber, balance, attemptedAmount) {
    super(`Insufficient funds in ${accountNumber}`);
    this.name = 'InsufficientFundsError';
    this.accountNumber = accountNumber;
    this.balance = balance;
    this.attemptedAmount = attemptedAmount;
    this.statusCode = 400;
  }
}

class AccountNotFoundError extends Error {
  constructor(accountNumber) {
    super(`Account ${accountNumber} not found`);
    this.name = 'AccountNotFoundError';
    this.accountNumber = accountNumber;
    this.statusCode = 404;
  }
}

class InvalidTransactionError extends Error {
  constructor(reason, transactionData) {
    super(`Invalid transaction: ${reason}`);
    this.name = 'InvalidTransactionError';
    this.reason = reason;
    this.transactionData = transactionData;
    this.statusCode = 400;
  }
}

// Using custom errors in banking operations
class BankingService {
  constructor() {
    this.accounts = new Map([
      ['ACC123456', { balance: 1000 }],
      ['ACC789012', { balance: 5000 }]
    ]);
  }

  async transfer(from, to, amount) {
    // Validation errors
    if (amount <= 0) {
      throw new InvalidTransactionError(
        'Amount must be positive',
        { from, to, amount }
      );
    }

    // Account not found error
    if (!this.accounts.has(from)) {
      throw new AccountNotFoundError(from);
    }

    const fromAccount = this.accounts.get(from);

    // Business logic error
    if (fromAccount.balance < amount) {
      throw new InsufficientFundsError(from, fromAccount.balance, amount);
    }

    // Perform transfer
    fromAccount.balance -= amount;
    this.accounts.get(to).balance += amount;

    return { success: true, newBalance: fromAccount.balance };
  }
}

// Error handling with custom errors
const bank = new BankingService();

(async () => {
  try {
    await bank.transfer('ACC123456', 'ACC789012', 2000);
  } catch (err) {
    if (err instanceof InsufficientFundsError) {
      console.error(`❌ ${err.message}`);
      console.error(`   Balance: $${err.balance} | Attempted: $${err.attemptedAmount}`);
    } else if (err instanceof AccountNotFoundError) {
      console.error(`❌ ${err.message}`);
    } else {
      console.error('❌ Unexpected error:', err);
    }
  }
})();

🔑 Key Takeaways

  • Error-first callbacks: (err, result) => {} pattern
  • Always check errors: Never ignore error parameter
  • Try-catch: For sync code and async/await
  • Custom errors: Extend Error for domain-specific errors
  • Error types: Operational (recoverable) vs programmer (bugs)

✅ DO's

  • Always check error parameter in callbacks
  • Use try-catch with async/await
  • Create custom error classes for business logic
  • Include contextual information in errors
  • Handle promise rejections (.catch or try-catch)

❌ DON'Ts

  • Don't swallow errors silently (empty catch blocks)
  • Don't use try-catch with callback-based async code
  • Don't throw strings (always throw Error objects)
  • Don't forget error handling in promise chains
  • Don't expose sensitive information in error messages

const fsPromises = require('fs/promises');
const path = require('path');

// Banking audit log system using fs.promises
class AuditLogger {
  constructor(logDir) {
    this.logDir = logDir;
  }

  async initialize() {
    // Create log directory if it doesn't exist
    try {
      await fsPromises.access(this.logDir);
    } catch {
      await fsPromises.mkdir(this.logDir, { recursive: true });
      console.log(`📁 Created log directory: ${this.logDir}`);
    }
  }

  async logTransaction(transaction) {
    const today = new Date().toISOString().split('T')[0]; // YYYY-MM-DD
    const logFile = path.join(this.logDir, `transactions-${today}.log`);

    const logEntry = JSON.stringify({
      timestamp: new Date().toISOString(),
      ...transaction
    }) + '\n';

    // Append to log file (creates if doesn't exist)
    await fsPromises.appendFile(logFile, logEntry, 'utf8');
    console.log(`✅ Logged transaction: ${transaction.txnId}`);
  }

  async getTransactions(date) {
    const logFile = path.join(this.logDir, `transactions-${date}.log`);

    try {
      const content = await fsPromises.readFile(logFile, 'utf8');
      const lines = content.trim().split('\n');
      return lines.map(line => JSON.parse(line));
    } catch (err) {
      if (err.code === 'ENOENT') {
        return []; // No transactions for this date
      }
      throw err;
    }
  }

  async listLogFiles() {
    const files = await fsPromises.readdir(this.logDir);
    return files.filter(file => file.endsWith('.log'));
  }
}

// Usage example
(async () => {
  const logger = new AuditLogger('./logs');
  await logger.initialize();

  // Log multiple transactions
  await logger.logTransaction({
    txnId: 'TXN001',
    from: 'ACC123456',
    to: 'ACC789012',
    amount: 5000,
    status: 'SUCCESS'
  });

  // Retrieve today's transactions
  const today = new Date().toISOString().split('T')[0];
  const transactions = await logger.getTransactions(today);
  console.log(`📊 Total transactions today: ${transactions.length}`);

  // List all log files
  const logFiles = await logger.listLogFiles();
  console.log(`📁 Log files: ${logFiles.join(', ')}`);
})();

  • fs.promises: Modern promise-based API (recommended)
  • Avoid sync methods: Block event loop in production
  • Use streams: For large files (>100MB)
  • Error handling: Check ENOENT, EACCES error codes

✅ DO's

  • Use streams for large files (>100MB)

  • Use path.join() for cross-platform paths

const http = require('http');

// In-memory account database
const accounts = new Map([
  ['ACC123456', { accountNumber: 'ACC123456', balance: 10000, name: 'John Doe' }],
  ['ACC789012', { accountNumber: 'ACC789012', balance: 5000, name: 'Jane Smith' }]
]);

// Helper: Parse JSON body
const parseBody = (req) => {
  return new Promise((resolve, reject) => {
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', () => {
      try {
        resolve(JSON.parse(body));
      } catch (err) {
        reject(new Error('Invalid JSON'));
      }
    });
  });
};

// Banking REST API
const server = http.createServer(async (req, res) => {
  const { pathname } = new URL(req.url, `http://${req.headers.host}`);
  const method = req.method;

  res.setHeader('Content-Type', 'application/json');

  try {
    // GET /api/accounts - List all accounts
    if (pathname === '/api/accounts' && method === 'GET') {
      res.writeHead(200);
      res.end(JSON.stringify({
        accounts: Array.from(accounts.values())
      }));
    }

    // GET /api/accounts/:id - Get account by ID
    else if (pathname.match(/^\/api\/accounts\/[A-Z0-9]+$/) && method === 'GET') {
      const accountNumber = pathname.split('/')[3];
      const account = accounts.get(accountNumber);

      if (account) {
        res.writeHead(200);
        res.end(JSON.stringify(account));
      } else {
        res.writeHead(404);
        res.end(JSON.stringify({ error: 'Account not found' }));
      }
    }

    // POST /api/accounts - Create new account
    else if (pathname === '/api/accounts' && method === 'POST') {
      const body = await parseBody(req);
      const { accountNumber, name, initialBalance } = body;

      if (accounts.has(accountNumber)) {
        res.writeHead(400);
        res.end(JSON.stringify({ error: 'Account already exists' }));
        return;
      }

      const newAccount = {
        accountNumber,
        name,
        balance: initialBalance || 0,
        createdAt: new Date().toISOString()
      };

      accounts.set(accountNumber, newAccount);

      res.writeHead(201);
      res.end(JSON.stringify(newAccount));
    }

    // POST /api/transfer - Transfer funds
    else if (pathname === '/api/transfer' && method === 'POST') {
      const { from, to, amount } = await parseBody(req);

      const fromAccount = accounts.get(from);
      const toAccount = accounts.get(to);

      if (!fromAccount || !toAccount) {
        res.writeHead(404);
        res.end(JSON.stringify({ error: 'Account not found' }));
        return;
      }

      if (fromAccount.balance < amount) {
        res.writeHead(400);
        res.end(JSON.stringify({ error: 'Insufficient funds' }));
        return;
      }

      // Perform transfer
      fromAccount.balance -= amount;
      toAccount.balance += amount;

      res.writeHead(200);
      res.end(JSON.stringify({
        success: true,
        txnId: 'TXN-' + Date.now(),
        from: { account: from, newBalance: fromAccount.balance },
        to: { account: to, newBalance: toAccount.balance }
      }));
    }

    else {
      res.writeHead(404);
      res.end(JSON.stringify({ error: 'Route not found' }));
    }

  } catch (err) {
    res.writeHead(500);
    res.end(JSON.stringify({ error: err.message }));
  }
});
